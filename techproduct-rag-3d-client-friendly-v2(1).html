02 / Problem Space

Why Traditional Search
Fails

❌
Keyword-Based Only
Exact match search misses synonyms, intent, and context. "Gaming laptop" won't find "portable PC for Valorant".
❌
No Intent Understanding
Traditional engines parse words, not meaning. Complex queries like "best budget option for a college student" return noise.
✓
Semantic Understanding
Vector embeddings capture the meaning of text, matching products by concept rather than exact terms.
✓
Context-Aware Responses
The AI composes tailored answers combining retrieved specs, prices, and recommendations into one coherent response.
03 / How It Works

System Architecture
Explained Simply

This section shows how the chatbot finds the right product information and turns it into a helpful answer for the user.

"The user asks → the system searches the product catalog → AI writes the final answer."
The image above shows the working chatbot. A user can ask questions like product prices, laptop suggestions, comparisons, or specs. Instead of guessing, the system first checks the product catalog and then gives a clear response based on the available data.
💬

User asks a question

The customer types a normal question, for example: “Which HP laptop is suitable for programming?”

📦

Product data is checked

The system looks inside the uploaded product catalog to find matching laptops, prices, specs, and details.

🔍

Best matches are selected

It brings forward only the most relevant products instead of showing random or unrelated results.

🤖

AI creates the answer

The chatbot explains the results in simple language, so the user gets a helpful recommendation quickly.
